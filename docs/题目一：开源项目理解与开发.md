# 题目一：开源项目理解与开发

> 基于「暴躁教授论文陪读」(mad-professor-public) 的二次开发方案

---

## 一、原项目架构精读

### 1.1 处理流水线（8阶段串行）

```
PDF
 ↓ pdf2md        — MinerU 提取 PDF → Markdown
 ↓ md2json       — MarkdownProcessor 结构化解析
 ↓ json_process  — JsonProcessor 清洗
 ↓ tiling        — TilingProcessor 分块
 ↓ translate     — TranslateProcessor 中英翻译（DeepSeek API）
 ↓ md_restore    — RestoreProcessor 还原为双语 Markdown
 ↓ extra_info    — ExtraInfoProcessor 章节摘要 + 问题 + 公式解析
 ↓ rag           — RagProcessor → FAISS 向量库 + 树结构 JSON
```

断点续跑机制：每个阶段检测输出文件是否已存在，存在则跳过。

### 1.2 核心模块分析

| 模块 | 文件 | 功能 | 亮点 | 局限 |
|---|---|---|---|---|
| AI 决策路由 | `ai_router_prompt.txt` | 一次 LLM 调用决定 emotion + function + query | 情绪感知 + Query 改写统一处理 | 每次问答多一次 LLM round-trip |
| 4 策略回答 | `AI_professor_chat.py` | direct / page_content / macro / rag | 分层检索，逐级精准 | RAG 阈值 0.6/0.65 硬编码 |
| TTS 语音 | `TTS_manager.py` | MiniMax speech-02-turbo，流式 PCM | 按句子流式播放 + cancel + 情绪映射 | voice_id 硬编码为 `leidianjiangjun` |
| STT 语音输入 | `voice_input.py` | RealtimeSTT + Whisper large-v2 | 实时 VAD、设备热切换 | **强依赖 CUDA，Mac 无法运行** |
| RAG 检索 | `rag_retriever.py` | FAISS + BGE-M3 embedding + 路径树定位 | 精确定位原文段落 + 滚动跳转 | 需 GPU 加载 BGE-M3 |
| 人设 prompt | `prompt/` | 可莉 / 雷电将军两套独立 txt | 零代码切换角色 | 无 UI 切换入口 |
| 数据管理 | `data_manager.py` | 论文索引 + 内容加载 | papers_index.json 全局索引 | — |
| 桌面 UI | `ui/` + PyQt6 | 分屏界面、Markdown 渲染、聊天气泡 | 功能完整 | 重量级桌面框架 |

### 1.3 技术栈依赖

```
Python 3.10 + CUDA + conda
├── MinerU (magic-pdf)     — PDF 解析（重量级，需要单独下载模型）
├── RealtimeSTT            — 语音识别（需要 Whisper + CUDA）
├── PyQt6 + PyQt6-WebEngine — 桌面 UI
├── FAISS-GPU              — 向量检索（只能 conda 安装）
├── sentence-transformers  — 嵌入模型（BGE-M3）
├── DeepSeek API           — LLM 对话（在线）
└── MiniMax API            — TTS 语音合成（在线）
```

### 1.4 发现的真实痛点

1. **环境地狱**：CUDA + conda + faiss-gpu + MinerU + RealtimeSTT，Mac/无GPU 完全跑不起来
2. **人设/声音切换无 UI**：README 明写"只能手动改代码"
3. **外放回声**：AI 声音被麦克风二次录入，只能戴耳机
4. **TTS voice_id 硬编码**：换人设必须改 `TTS_manager.py` 源码
5. **RAG 阈值硬编码**：0.6 / 0.65 固定值，不同文档效果差异大
6. **仅支持论文结构 PDF**：非学术文档直接报错（README 已知问题）
7. **对话历史仅 10 条**：长对话丢失上下文
8. **无引导性问题推荐**：不能像 ChatGPT 那样给出"你可能还想问"

## 三、推荐方案：方向 C —「暴躁教授看B站」

### 3.1 核心创意

把"学术论文 AI 陪读"的模块**剥离重组**为"B站视频 AI 评论员"：

> 输入一条 B站视频链接 → 自动下载字幕 → 建立语义索引 → 用可莉/雷电将军人设和你讨论视频内容 → 教授会根据问题情绪路由决定是"温柔解答"还是"暴怒吐槽"

### 3.2 推荐理由

1. **契合"折腾精神"**：将论文陪读模块拆出来重组成 B站视频评论员，创意重组而非简单修改
2. **完美利用已有工作**：已下载 3 个林亦视频字幕（图灵测试大逃杀、三大AI美国开战），直接做演示素材
3. **大幅降低环境依赖**：不需要 CUDA / MinerU / PyQt6 / FAISS-GPU，`pip install` 即用
4. **核心代码复用率高**：LLMClient、人设 prompt、情绪路由逻辑直接复用原项目，证明"理解开源项目"
5. **3分钟视频好拍**：终端 → 输入链接 → 下载字幕 → 可莉口吻问答 → 教授破防 → 天然笑点
6. **趣味性拉满**：林亦 HR 收到一个"用可莉口吻评论林亦自己视频"的工具，不可能不乐

### 3.3 核心代码复用清单

| 原项目文件 | 复用方式 |
|---|---|
| `config.py` → `LLMClient` | **直接复用**，去掉 EmbeddingModel 和 TTS 配置 |
| `AI_professor_chat.py` → 情绪路由 + 策略分发 | **核心复用**，简化为 direct + rag 两策略 |
| `prompt/ai_character_prompt_*.txt` | **直接复制** |
| `prompt/ai_router_prompt.txt` | **复用并微调**（去掉 page_content 和 macro 策略） |
| `prompt/ai_explain_prompt.txt` | **复用并微调**（"论文" → "视频"） |
| `rag_retriever.py` | **重写为轻量版**（FAISS-GPU → sentence-transformers CPU） |
| `TTS_manager.py` | **可选复用**（如有 MiniMax key 可保留语音播报） |

### 3.4 不需要的重型依赖

| 去掉的 | 原因 |
|---|---|
| MinerU (magic-pdf) | 不处理 PDF |
| RealtimeSTT + Whisper | 不需要语音输入 |
| PyQt6 + PyQt6-WebEngine | CLI 工具不需要桌面 UI |
| FAISS-GPU + conda | 用 `faiss-cpu` 或纯 `sentence-transformers` |
| CUDA | 全部 CPU 运行 |

### 3.5 技术架构

```
B站视频 URL
     ↓
download_bilibili_subs.py（已有！）
     ↓
字幕文本 → 按段落/时间窗分割 → sentence-transformers (all-MiniLM-L6-v2, CPU)
     ↓
内存 FAISS 向量库（faiss-cpu，无需 conda）
     ↓
用户提问 → 情绪路由（复用 ai_router_prompt.txt）→ 向量检索 Top-K → 拼入上下文
     ↓
LLMClient (DeepSeek API) + 人设 prompt
     ↓
流式终端彩色输出（可选：MiniMax TTS 语音播报）
```

### 3.6 趣味功能设计

- **多人设切换**：启动时 / 运行中可切换可莉 ↔ 雷电将军（性格截然相反）
- **教授主动提问**：每隔几轮对话，教授主动考你："你看完视频了吗？回答我几个问题！"
- **暴怒触发机制**：问到视频里没提到的内容，教授直接破防开骂
- **字幕吐槽模式**：检测到 B站 AI 字幕明显错误时，教授自动吐槽
- **视频章节导航**：根据字幕时间戳告诉你 "去看视频 XX:XX 那段"
