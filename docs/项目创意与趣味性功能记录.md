# 暴躁教授看B站 —— 核心创意与趣味性功能讨论记录

> 本文档记录了关于“暴躁教授”项目在面试场景下，为了凸显“独立开发者的幽默感和折腾精神”而确立的全部核心创意、特效构想及技术拆解方案。

## 🎯 核心目标：把“逗笑评审”作为最高技术优先级
技术架构做的好是敲门砖，但在本次二创任务中，**“趣味性、幽默感、反差萌”必须刻在每一个交互的 DNA 里**。不仅前端要好看，提示词和场景也要充满“戏”。

---

## 🎭 1. 角色人设与行为逻辑 (Persona & Behavior)
不只是干巴巴的问答，而是带有情绪起伏的动态 NPC：
- **无厘头学术风**：在日常吐槽中突然引用硬核学术名词，强行解释极其尴尬的场景（比如用热力学第二定律解释为什么视频这么无聊）。
- **老派训人模式**：捕捉到用户输入简短或不用心时，触发“恨铁不成钢”的专属暴躁状态，伴随专属的文案。
- **自黑与打破第四面墙**：教授在极度暴躁后可能会自我反省：“我的 Prompt 设定是不是太严格了？”，或者嘲讽原来的 PyQt 原版框架多难用。

---

## 🎨 2. 也是最高光的部分：工具调用驱动前端动画 (Generative UI)
不再是单调的文本回复，利用大模型的 `Tool Calling` 能力，前端拦截工具指令，渲染**夸张的 HTML 动态组件**。

| 触发场景 | LLM 调用的工具 (Tool) | 前端 HTML 对应呈现效果 (可视化幽默) |
| :--- | :--- | :--- |
| **检测到愚蠢问题** | `trigger_anger_meter(level)` | 屏幕边缘泛红闪烁，弹出一个**“愤怒值爆表”**的夸张动态进度条，甚至震动屏幕。 |
| **讲了烂梗/查不到资料** | `trigger_awkward_silence(seconds)` | 出现一个**“尴尬冷场”计时器**，倒数 3 秒后，屏幕上蹦出一个硕大的“尴尬大比拼得分：-99”。 |
| **用户表现优异/答对问题** | `trigger_confetti()` / `play_guitar()` | 经典的**满屏彩带烟花（Confetti）**发射，或者弹出一个教授弹吉他庆祝的像素/SVG 动效小面板。 |
| **对用户回答打分** | `show_scoring_board(score)` | 其实不是文字打分，而是在聊天流中直接渲染一个 3D/CSS 动画的**打分牌挥舞特效**。 |

---

## 🛍️ 3. 角色构建体系与“性格大厅” (Persona Market)
为了让项目的延展性拉满，构建一个**“角色插件生态 / 角色大厅”**：
- **可插拔的角色包**：不仅有“暴躁教授”，可以预置“温柔学妹”、“键盘侠”、“迷糊助教”等不同的人设数据包（包含对应的 Prompt、专门的动画工具权限）。
- **用户自创**：允许将特定的打分板、特定的话术打包成一个独立配置（构想中的功能，用于展现项目具备开放生态的设计理念）。

---

## 📝 4. 挑战模式与互动出题 (Interactive Quizzes)
传统的 RAG 都是用户问、AI 答，我们要做一个反转：**教授主动考你！**
- **随机掉落的小测验**：论文/视频内容解析到一半，教授会突然暂停：“你看懂了吗？考考你！”
- **富文本题目渲染**：通过工具调用，在对话框里直接渲染出完整的交互组件（单选题、完型填空、阅读简答等）。
- **奖励闭环**：答对了不仅解锁后续剧情，还会触发上面的“Confetti 烟花”成就感拉满；答错了则触发“愤怒值动画”和一堆刻薄的训斥。

---

## 🔧 5. 技术重构与独立验证思路说明
抛弃原版难用的 Python + PyQt，拥抱 TypeScript + React + Serverless 架构，在开发路径上我们确立了**“积木式独立验证”**的策略：
1. 先验证最底层的**基础文本生成与路由逻辑**。
2. 验证**工具调用 UI (Generative UI)** 的渲染（比如调一个组件出来放烟花）。
3. 验证**Markdown 渲染与多模态**支持能力。
4. 验证**语音交互/图像生成**（如有余力作为彩蛋加进去）。
5. 最后把这些验证通过的模块串联组装，保证核心“趣味性”万无一失。
